# Coreflow

Coreflow is organized as a monorepo with a separate frontend and backend:

- `frontend/`: Next.js SaaS client with Supabase auth and the current habits module
- `backend/`: Spring Boot backend scaffold for the future domain API

## Working Directories

- Frontend commands should be run from `frontend/`
- Backend commands should be run from `backend/`

## Current State

The frontend remains functional with Supabase while the backend structure is being prepared for a gradual migration to a dedicated Java API.
